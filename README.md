Monorepo of shareable linter configurations used by the Inclusive Design Research Centre.

## Development

This monorepo uses the shareable linter configurations in each package to lint itself. To lint the entire monorepo, run:

```bash
npm run lint
```

Tests for each linter are in the respective `/packages/*/test` directory. To execute tests for all packages from the
root directory, run:

```bash
npm run test
```

To run tests for a specific package from the root directory, run:

```bash
npm run test --workspace packages/eslint-config
```

## Releasing

This repository uses [release-please](https://github.com/googleapis/release-please-action) to generate release pull
requests for each package based on [semantic versioning](https://semver.org/). A GitHub release can be generated by
merging a release pull request such as [this one](https://github.com/inclusive-design/linter-configs/pull/4).

To publish the package to the [@inclusive-design npm organization](https://www.npmjs.com/org/inclusive-design), use the
npm CLI to log in:

```bash
npm login
```

Then, to publish the desired package (in this example, `@inclusive-design/eslint-config`), run:

```bash
npm publish --access public --workspace packages/eslint-config
```
